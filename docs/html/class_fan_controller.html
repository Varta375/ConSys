Подробное описание

Класс для управления вентиляторами и их мониторинга для ноутбуков Acer Nitro 5 AN515-45 (Ryzen 5 5600H RTX 3060).

Управляет оборотами через запись в реестр HKLM\SOFTWARE\OEM\NitroSense\FanControl и отправку структурированных команд службе PSsvc через именованный канал Идея реализации базируется на решениях проекте NitroSensual от KRWCLASSIC

См. также
https://github.com/KRWCLASSIC/NitroSensual

◆ ApplyChangesViaPipe()

bool FanController::ApplyChangesViaPipe ( FanType fanType,
DWORD percent )

Отправка команды на применение скорости вентилятора через именованный канал

Аргументы
fanTypeТип целевого вентилятора (CPU или GPU)
percentСкорость вращения в процентах (0-100)%

Упаковывает параметры в 64-битную нагрузку, формирует структуру FanCommandPacket (15 байт) и отправляет её в канал \.\pipe\PredatorSense_service_namedpipe. Считывает 9-байтовый ответ

Возвращает
true, если пакет был успешно записан в канал.
Исключения
std::exceptionОшибки логируются в std::cerr, а в качестве результата возвращается false

◆ SetCpuFanSpeed()

bool FanController::SetCpuFanSpeed ( DWORD percentage)

Установка скорости оборотов вентилятора процессора (CPU).

Аргументы
percentageСкорость вращения в процентах (0-100)%

Записывает значение в реестр под именем "CPUFanPercentage" и мгновенно применяет его, отправляя команду через именованный канал

Возвращает
true, если значение успешно записано и передано службе; иначе false
Предупреждения
Настройка применится аппаратно только в том случае, если в NitroSense выбран режим работы "Custom"
Исключения
std::exceptionОшибки логируются в std::cerr, а в качестве результата возвращается false

◆ SetFanMode()

bool FanController::SetFanMode ( int mode)

Установка режима работы вентиляторов ноутбука (Custom, Max).

Аргументы
modeРежим работы: 1 - Custom, 2 - Max

Изменяет значение "CurrentFanMode" в реестре и отправляет пакеты инициализации выбранного режима обоим вентиляторам через именованный канал

Возвращает
true, если режим успешно изменен; false при передаче недопустимого значения режима
Исключения
std::exceptionОшибки логируются в std::cerr, а в качестве результата возвращается false
