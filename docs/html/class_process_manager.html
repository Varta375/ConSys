Подробное описание

Класс для создания снимка действующих процессов системы

Программа использует библиотеку Windows.h для создания снимка системы. Позволяет получать список процессов и обновлять его при необходимости. Кроме того, существует возможность приостановить процесс (плавно, с таймаутом) из списка.

См. также
https://learn.microsoft.com/ru-ru/windows/uwp/get-started/universal-application-platform-guide

◆ Refresh()

const std::vector< ProcessInfo > & ProcessManager::Refresh ( )

Отвечает за получение списка процессов и его обновление

Метод создает снэпшот состояния системы путем вызова CreateToolhelp32Snapshot с флагом TH32CS_SNAPPROCESS и ID 0 (сохранение всех процессов). После этого пробегается по полученному снэпшоту, фильтрует его и сохраняет во внутренний вектор Processes. Работа кода строится на Windows API (библиотека Tool Help Library).

Возвращает
Вектор процессов системы, прошедших фильтрацию. В случае критического сбоя возвращается пустой или частично заполненный вектор.
Исключения
std::exceptionМетод не выбрасывает данное исключение наружу. Любые возникающие стандартные исключения перехватываются внутри, выбрасываются с помощью std::cerr.

◆ StopProcess()

bool ProcessManager::StopProcess ( DWORD pid)

Отвечает за двухэтапное завершение процесса (мягкое и жесткое закрытие, с таймаутом).

Сначала ищется PID в векторе Processes. Если он нашелся, то открывается его дескриптор. PROCESS_TERMINATE отвечает за принудительное завершение процесса. Срабатывает только в том случае, если не сработает мягкое закрытие WM_CLOSE.

Аргументы
pidЗначение ID процесса для его приостановки
См. также
https://learn.microsoft.com/ru-ru/windows/win32/api/processthreadsapi/nf-processthreadsapi-openprocess
https://learn.microsoft.com/ru-ru/windows/win32/api/winuser/nf-winuser-enumwindows
https://learn.microsoft.com/ru-ru/windows/win32/api/synchapi/nf-synchapi-waitforsingleobject
Возвращает
Флаг, указывающий на успех (true) или неудачу (false) приостановки процесса.
Исключения
std::exceptionМетод не выбрасывает исключений наружу. Любые ошибки или исключения при обработке процесса перехватываются внутри, открытый дескриптор безопасно закрывается, а метод возвращает false.
